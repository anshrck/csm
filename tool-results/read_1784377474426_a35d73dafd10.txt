    94→    94→
    95→    95→DEMAND STATE MACHINE (agents 1, 5, 6, 7, 8):
    96→    96→  NEW → (assign + start review) → UNDER_REVIEW → (quote | reject | redirect)
    97→    97→  QUOTED requires quoteApprovedByCmLeader=true (CM Leader gate) → (customer accept | decline)
    98→    98→  ACCEPTED → (create change + hand to CE) → IN_CHANGE → (mark fulfilled) → FULFILLED → (close) → CLOSED
    99→    99→  Rejection/redirect are terminal. Each transition emits a DemandEvent.
   100→   100→
   101→   101→CHANGE STATE MACHINE (agents 4, 5):
   102→   102→  REQUESTED → ASSESSMENT → PLANNING → APPROVED → IMPLEMENTATION → VERIFICATION → CLOSED
   103→   103→  (REJECTED terminal). Closing a change with originDemandId sets that demand to FULFILLED.
   104→   104→
   105→   105→NOTIFICATIONS: created on key events (DemandQuoted, DemandAccepted, DemandFulfilled, DemandRejected,
   106→   106→  DemandCreated, SlaBreached, SlaWarning, ChangeClosed). Target userId depends on event.
