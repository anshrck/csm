    85→  - All API routes use `export const runtime = 'nodejs';`
    86→  - Auth: `import { getSession, requireRole } from '@/lib/auth'`; `import { db } from '@/lib/db'`.
    87→  - Return NextResponse.json(...) with proper status codes.

    91→  - Dates: return ISO strings (Prisma returns Date objects; call .toISOString()).
    92→  - Role checks: SERVICE_CUSTOMER scoped to own orgNode; SCM_WORKER scoped to assigned demands;
    93→    CM_LEADER sees all tenant; SERVICE_OWNER sees services they own.
    94→
    95→DEMAND STATE MACHINE (agents 1, 5, 6, 7, 8):
    96→  NEW → (assign + start review) → UNDER_REVIEW → (quote | reject | redirect)
    97→  QUOTED requires quoteApprovedByCmLeader=true (CM Leader gate) → (customer accept | decline)
    98→  ACCEPTED → (create change + hand to CE) → IN_CHANGE → (mark fulfilled) → FULFILLED → (close) → CLOSED
    99→  Rejection/redirect are terminal. Each transition emits a DemandEvent.
   100→
   101→CHANGE STATE MACHINE (agents 4, 5):
   102→  REQUESTED → ASSESSMENT → PLANNING → APPROVED → IMPLEMENTATION → VERIFICATION → CLOSED
   103→  (REJECTED terminal). Closing a change with originDemandId sets that demand to FULFILLED.
   104→
   105→NOTIFICATIONS: created on key events (DemandQuoted, DemandAccepted, DemandFulfilled, DemandRejected,
   106→  DemandCreated, SlaBreached, SlaWarning, ChangeClosed). Target userId depends on event.
