  1202→## Routing design notes
  1203→
  1204→### URL ⇄ store feedback loop
  1205→The catch-all page uses a `pushedRef` to break the cycle:
  1206→1. User clicks sidebar nav → `navigate(view)` → store updates + navTick bumps.
  1207→2. Catch-all's store→URL effect fires → `pushedRef = true` → `router.push(expectedPath)`.
  1208→3. URL changes → Next.js re-renders with new `slug` prop.
  1209→4. URL→store effect fires → sees `pushedRef === true` → clears ref + skips re-sync.
  1210→
  1211→Browser back/forward:
  1212→1. URL changes (no `router.push` from us) → Next.js re-renders with new `slug`.
  1213→2. URL→store effect fires → `pushedRef === false` → calls `syncFromUrl(slug)` → store updates.
  1214→3. Store→URL effect fires → expected path already matches pathname → no push.
  1215→
  1216→This avoids infinite loops while keeping URL and store in sync.
  1217→
  1218→### Next.js 16 `params` is a Promise
  1219→The catch-all page is a client component. In Next.js 16, `params` is a Promise and must be unwrapped with `React.use()` (or `await` in a server component) before accessing its properties.
