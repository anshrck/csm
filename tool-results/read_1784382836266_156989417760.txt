    96‚Üí  NEW ‚Üí (assign + start review) ‚Üí UNDER_REVIEW ‚Üí (quote | reject | redirect)
    97‚Üí  QUOTED requires quoteApprovedByCmLeader=true (CM Leader gate) ‚Üí (customer accept | decline)
    98‚Üí  ACCEPTED ‚Üí (create change + hand to CE) ‚Üí IN_CHANGE ‚Üí (mark fulfilled) ‚Üí FULFILLED ‚Üí (close) ‚Üí CLOSED
    99‚Üí  Rejection/redirect are terminal. Each transition emits a DemandEvent.

   103‚Üí  (REJECTED terminal). Closing a change with originDemandId sets that demand to FULFILLED.

  1202‚Üí## Routing design notes
  1203‚Üí
  1204‚Üí### URL ‚áÑ store feedback loop
  1205‚ÜíThe catch-all page uses a `pushedRef` to break the cycle:
  1206‚Üí1. User clicks sidebar nav ‚Üí `navigate(view)` ‚Üí store updates + navTick bumps.
  1207‚Üí2. Catch-all's store‚ÜíURL effect fires ‚Üí `pushedRef = true` ‚Üí `router.push(expectedPath)`.
  1208‚Üí3. URL changes ‚Üí Next.js re-renders with new `slug` prop.
  1209‚Üí4. URL‚Üístore effect fires ‚Üí sees `pushedRef === true` ‚Üí clears ref + skips re-sync.
  1210‚Üí
  1211‚ÜíBrowser back/forward:
  1212‚Üí1. URL changes (no `router.push` from us) ‚Üí Next.js re-renders with new `slug`.
  1213‚Üí2. URL‚Üístore effect fires ‚Üí `pushedRef === false` ‚Üí calls `syncFromUrl(slug)` ‚Üí store updates.
  1214‚Üí3. Store‚ÜíURL effect fires ‚Üí expected path already matches pathname ‚Üí no push.
  1215‚Üí
  1216‚ÜíThis avoids infinite loops while keeping URL and store in sync.
  1217‚Üí
  1218‚Üí### Next.js 16 `params` is a Promise
  1219‚ÜíThe catch-all page is a client component. In Next.js 16, `params` is a Promise and must be unwrapped with `React.use()` (or `await` in a server component) before accessing its properties. The page unwraps once at the top: `const { slug } = use(params);` ‚Äî then uses `slug` everywhere. Caught this from the dev.log error on first test.
  1220‚Üí
  1221‚Üí### Detail views reuse list paths
  1222‚Üí- `demand-detail` ‚Üí `/demands/<id>`
  1223‚Üí- `change-detail` ‚Üí `/changes/<id>`
  1224‚Üí- `ticket-detail` ‚Üí `/tickets/<id>`
  1225‚Üí
  1226‚Üí`pathToView()` uses `DETAIL_VIEWS` to convert URL+id to the detail ViewKey; `viewToPath()` appends the id segment for the three detail ViewKeys.
